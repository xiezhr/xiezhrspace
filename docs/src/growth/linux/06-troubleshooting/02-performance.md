## 性能快速排查

> 服务卡顿？先看资源够不够。

### 排查顺序

| 检查项 | 命令 | 指标 |
|--------|------|------|
| CPU | `top` | %CPU > 80% 要注意 |
| 内存 | `free -h` | 可用内存 < 10% 要注意 |
| 磁盘 | `df -h` | 使用率 > 80% 要清理 |
| IO | `iostat` | 磁盘读写是否饱和 |

### 常见场景

- 服务响应慢 → `top` 看 CPU → `free` 看内存 → `df` 看磁盘
- 日志写不进 → `df -h` 发现磁盘满了
